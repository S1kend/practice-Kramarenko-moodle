Интернет — глобальная сеть компьютеров, обеспечивающая передачу данных между устройствами.
Веб (World Wide Web) — сервис внутри интернета, позволяющий получать доступ к сайтам через браузер.
Браузер — программа-клиент, которая отправляет запросы и отображает веб-страницы.
Сервер — компьютер, который обрабатывает запросы клиента и отправляет ему данные.
HTTP — протокол передачи данных между клиентом и сервером.
HTTPS — защищённая версия HTTP с шифрованием данных.
DNS — система, преобразующая доменное имя сайта в IP-адрес.
HTML — язык разметки, используемый для создания структуры веб-страницы.
Тег — элемент HTML, который определяет структуру и содержание страницы (например, <p>, <h1>).
Атрибут — дополнительная информация внутри тега HTML (например, href, src, alt).
CSS (Cascading Style Sheets) — язык стилей, который отвечает за внешний вид веб‑страницы: цвета, размеры, расположение элементов.
Селектор — способ указать, к каким элементам HTML применять CSS‑правила (например, h1 — по тегу, .class — по классу, #id — по идентификатору).
Блочная модель (box model) — концепция CSS, согласно которой каждый элемент представляет собой прямоугольник, состоящий из содержимого, внутренних отступов (padding), границы (border) и внешних отступов (margin).
Flexbox — модуль CSS для гибкого позиционирования элементов внутри контейнера, упрощающий выравнивание и распределение пространства.
Медиазапрос (media query) — инструмент CSS, позволяющий применять стили в зависимости от характеристик устройства (ширина экрана, ориентация и т.д.).
JavaScript (JS) — язык программирования, который добавляет веб‑страницам интерактивность: обработку кликов, изменение содержимого, работу с формами и динамику.
Переменная — именованная ячейка для хранения данных; в JavaScript объявляется с помощью ключевых слов let или const.
Функция — блок кода, который можно многократно вызывать для выполнения определённой задачи (например, function sayHello() { alert("Привет!"); }).
Массив (array) — структура данных в JavaScript, которая хранит упорядоченный набор значений (элементов) и позволяет обращаться к ним по индексу.
Цикл (loop) — конструкция в JavaScript (например, for, while), позволяющая многократно выполнять один и тот же блок кода.
Событие (event) — действие пользователя или системы (клик, ввод текста, загрузка страницы), на которое можно запрограммировать ответ (например, атрибут onclick вызывает функцию при нажатии на элемент).
DOM (Document Object Model) — объектная модель документа, представление HTML‑страницы в виде дерева узлов, которое JavaScript может изменять (добавлять, удалять, менять содержимое и стили).
Git — система контроля версий, позволяющая отслеживать изменения в файлах, возвращаться к предыдущим версиям и работать над проектом в команде.
Репозиторий (repository) — хранилище проекта под управлением Git; может находиться локально на компьютере или удалённо (например, на GitHub).
Коммит (commit) — фиксация изменений в Git, сохранение текущего состояния файлов с комментарием (создаётся командой git commit -m "сообщение").
GitHub — онлайн‑платформа для хранения Git‑репозиториев, совместной разработки, отслеживания задач и публикации проектов.
Push — команда Git (git push), которая отправляет локальные коммиты в удалённый репозиторий (например, на GitHub).
Pull — команда Git (git pull), которая загружает изменения из удалённого репозитория и объединяет их с локальной версией проекта.
Ветка (branch) — изолированная линия разработки в Git, позволяющая параллельно вести изменения, не влияя на основную версию проекта.
Адаптивный дизайн (responsive design) — подход к вёрстке, при котором страница корректно отображается на разных устройствах и размерах экрана.
